## 🛠️ Methodology & Tools
I followed the industry-standard "Vulnerability Assessment" lifecycle:

2.  **Vulnerability Scanning:** Leveraged **OWASP ZAP** (Zed Attack Proxy) to perform automated active and passive scanning.
3.  **Risk Analysis:** Categorized findings based on severity levels (Medium, Low, Informational) using the **CWE (Common Weakness Enumeration)** framework.
4.  **Reporting:** Documentation of results and remediation strategies.

## 📊 Technical Findings Summary

### 1. Network Scan (Nmap)
The scan revealed several active ports, including:
* **Port 21 (FTP):** Potential vector for unauthorized access if misconfigured.
* **Port 1720/8010:** Identified as `tcpwrapped`, indicating firewall protection or specific service filtering.

### 2. Security Vulnerabilities (OWASP ZAP)
A total of **10 Alerts** were identified during the automated scan:

| Severity | Count | Key Vulnerabilities |
| :--- | :--- | :--- |
| 🟠 **Medium** | 5 | CSP Header Not Set, Missing Anti-clickjacking, Vulnerable JS Library |
| 🟡 **Low** | 3 | X-Content-Type-Options Missing, Server Version Leak |

---

## 🔍 Critical Vulnerabilities & Remediation

### A. Missing Security Headers (CSP & X-Frame-Options)
* **Risk:** High susceptibility to Cross-Site Scripting (XSS) and Clickjacking.
* **Solution:** Implement `Content-Security-Policy` and `X-Frame-Options: DENY` in HTTP response headers.

### B. Vulnerable Third-Party Libraries
* **Risk:** Outdated JavaScript libraries may contain known public exploits.
* **Solution:** Update all frontend libraries to the latest patched versions.

### C. Information Disclosure
* **Risk:** Revealing server versions (Apache Tomcat) helps attackers target specific exploits.
* **Solution:** Disable server signatures in the configuration settings.
